# SignFlow Bot ✍️📄

A high-performance asynchronous Python Telegram bot optimized for Render.com that applies transparent digital signature overrides onto PDF layout vectors and image canvases.

## 🚀 Step-by-Step Render Deployment Guide

### Step 1: Push Source Files Directly to your GitHub Root Directory
Ensure that all required structural execution files (`bot.py`, `handlers.py`, `database.py`, `config.py`, `signature_manager.py`, `pdf_signer.py`, `image_signer.py`, `utils.py`, `requirements.txt`, etc.) are placed immediately in the root repository folder. Do not nest them within sub-level project folder layouts:
```bash
git init
git add .
git commit -m "Initialize structural footprint package baseline for SignFlow Bot Engine"
git branch -M main
git remote add origin [https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git](https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git)
git push -u origin main
